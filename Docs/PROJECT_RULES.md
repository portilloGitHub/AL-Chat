# Project Organization Rules

This document outlines the organizational rules for the AL-Chat project.

## Folder Structure Rules

### 1. Test Folder (`Test/`)
**Rule:** All test-related items MUST go into the `Test/` folder.

- ✅ Test scripts (`test_*.py`)
- ✅ Test utilities and helpers
- ✅ Test configuration files
- ✅ Test data and fixtures
- ✅ Test documentation
- ✅ Test requirements

**See:** [Test/TEST_RULES.md](../Test/TEST_RULES.md) for detailed rules.

### 2. Documentation Folder (`Docs/`)
**Rule:** All `.md` documentation files go into `Docs/` folder.

- ✅ All markdown documentation files
- ✅ Setup guides
- ✅ Architecture documentation
- ✅ Contributing guidelines
- ❌ Exception: Session logs (go in `SessionLog/`)

### 3. Session Logs (`SessionLog/`)
**Rule:** Daily session logs are stored here.

- ✅ Daily log files: `session_YYYY-MM-DD.log`
- ✅ Auto-generated by session logger
- ❌ Do not manually create log files

### 4. Code Review (`CodeReview/`)
**Rule:** Code review notes and feedback go here.

- ✅ Review notes
- ✅ Feedback documents
- ✅ Code review checklists

### 5. Backend (`Backend/`)
**Rule:** Python backend code and configuration.

- ✅ Python source files
- ✅ `credentials/` - Credential management
- ✅ `service/` - Business logic services
- ✅ `requirements.txt`
- ✅ `.env` file (not committed to git)

### 6. Frontend (`Frontend/`)
**Rule:** React frontend code and configuration.

- ✅ React source files
- ✅ `package.json`
- ✅ Frontend configuration

## File Naming Conventions

- Test files: `test_<name>.py` in `Test/`
- Documentation: `*.md` in `Docs/`
- Session logs: `session_YYYY-MM-DD.log` in `SessionLog/`
- Credentials: `.env` in `Backend/` (not committed)

## Enforcement

- Code reviews should verify folder structure compliance
- New files should follow these rules
- Documentation should reference these rules
